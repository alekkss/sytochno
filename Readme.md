# Sutochno Parser — мониторинг посуточной аренды

Программа для автоматического сбора объявлений посуточной аренды с сайта sutochno.ru, извлечения данных о ценах, календаре занятости и характеристиках объектов с экспортом результатов в Excel.

---

## Что делает программа

Программа работает в три этапа:

**Этап 1 — Парсинг каталога.** Открывает браузер, переходит по ссылке на страницу поиска sutochno.ru (с уже настроенными фильтрами), обходит все страницы пагинации и собирает базовые данные каждого объявления: ID, название, цену за сутки, рейтинг, количество отзывов, площадь, количество гостей, адрес, ближайшее метро, наличие быстрого бронирования. Дедупликация по ID выполняется автоматически.

**Этап 2 — Парсинг карточек (гибридная стратегия).** Заходит в каждую карточку объявления и извлекает календарь занятости и цены на 60 дней через внутреннее API sutochno.ru (`getPricesAndAvailabilities`). Вместо 60 отдельных запросов программа использует гибридную стратегию:

1. **Перехват токена** — сессионный токен API перехватывается через `page.route("**/api/json/**")` при загрузке страницы карточки.
2. **Валидация токена** — тестовый запрос на 2 ночи проверяет работоспособность токена.
3. **Bulk-запрос цен** — один запрос на 60 ночей получает все цены сразу. API возвращает записи `detail[]` с типами `season_price` (сезонные цены с диапазонами дат) и `type=1` (единая базовая цена). Программа разворачивает сезонные цены в дневные и использует базовую цену как fallback.
4. **Определение занятости** — если `busy="unbusy"`, все дни свободны. Если `busy="busy"` — запускается скользящее окно: 60 запросов пакетами по 5 параллельно с паузой 0.5 секунды между пакетами. При ошибках `min_nights` программа адаптивно перебирает варианты (2, 3, 4, 5, 6, 7, 10, 14, 30 ночей).
5. **Retry** — при протухшем токене или массовых ошибках страница перезагружается и процесс повторяется с новым токеном.

Для ускорения обработки доступны параллельные вкладки (`MAX_TABS`) и параллельные прокси-браузеры (`USE_PROXY`). Каждый прокси-браузер дополнительно использует `MAX_TABS` вкладок.

**Этап 3 — Экспорт.** Формирует Excel-файл с таблицей, автофильтрами и кликабельными ссылками. Вы можете отсортировать объявления по цене, занятости или рейтингу и сравнить объекты между собой.

## Установка

### 1. Клонируйте или скачайте проект

Поместите папку `sutochno_parser` в удобное место на вашем компьютере.

### 2. Откройте терминал в папке проекта

```bash
cd путь/к/sutochno_parser
```

### 3. Создайте виртуальное окружение (рекомендуется)

```bash
python -m venv venv
```

Активируйте его:

**Windows:**
```bash
venv\Scripts\activate
```

**macOS / Linux:**
```bash
source venv/bin/activate
```

### 4. Установите зависимости

Основные зависимости:
```bash
pip install -e .
```

Для разработки (линтер, типы, тесты):
```bash
pip install -e ".[dev]"
```

```bash
playwright install chromium
```

Эта команда скачает Chromium (~150 МБ). Загрузка выполняется один раз.

## Запуск

### Основная команда

```bash
python -m src
```

### Что происходит при запуске

1. Программа загружает настройки из `.env` и проверяет обязательные переменные.
2. Открывается браузер Chromium со stealth-настройками (обход антибот-защиты).
3. Браузер переходит на указанную страницу поиска sutochno.ru.
4. Программа собирает базовые данные всех объявлений на каждой странице каталога.
5. При наличии пагинации — автоматически нажимает «Далее» и переходит на следующие страницы.
6. После обхода каталога — программа обогащает каждую карточку:
   - Загружает страницу карточки и перехватывает сессионный токен API через `page.route`.
   - Валидирует токен тестовым запросом.
   - Отправляет bulk-запрос на 60 ночей для получения всех цен (сезонные + базовая).
   - При `busy="busy"` запускает скользящее окно (пакетами по 5) для определения занятости каждого дня.
   - При ошибках адаптирует `min_nights` или перезагружает страницу с новым токеном.
   - Режим обработки выбирается автоматически: прокси+вкладки → вкладки → последовательно.
7. Собранные данные сохраняются в SQLite (`data/sutochno_listings.db`).
8. Результаты экспортируются в Excel (`data/sutochno_report.xlsx`).
9. Логи сохраняются в файл `logs/app.log` в формате JSON с ротацией (10 МБ × 5 копий).

### Альтернативный запуск (если установлена как пакет)

```bash
sutochno-parser
```

## Результат — Excel-файл

После успешного запуска в папке `data/` появится файл `sutochno_report.xlsx` со следующими столбцами:

| Столбец | Описание |
|---|---|
| ID объявления | Уникальный идентификатор объявления на sutochno.ru |
| Название | Оригинальное название из объявления |
| Цена (руб./сут.) | Цена за сутки в рублях |
| Рейтинг | Рейтинг объекта (например, 9.1) |
| Отзывы | Количество отзывов |
| Площадь (м²) | Площадь объекта |
| Гостей | Максимальное количество гостей |
| Адрес | Полный адрес объекта |
| Метро | Ближайшая станция метро с расстоянием |
| Быстрое бронирование | Наличие быстрого бронирования (Да/Нет) |
| Занятость (%) | Процент занятых дней из ближайших 60 |
| Календарь 60 дней | Строка из 0 и 1 на 60 дней (0 — свободен, 1 — занят) |
| Средняя цена (руб./сут.) | Средняя цена за сутки по свободным дням |
| Цены 60 дней (руб./сут.) | Цены за сутки через «;» на 60 дней (0 — день занят) |
| Ссылка | Прямая ссылка на объявление (кликабельная) |
| Дата снимка | Дата и время сбора данных |

### Как работать с файлом

1. Откройте файл в Excel или LibreOffice Calc.
2. Используйте автофильтр — выберите нужные параметры (цена, рейтинг, район).
3. Отсортируйте по столбцу **«Цена (руб./сут.)»** — увидите объекты от дешёвых к дорогим.
4. Отсортируйте по столбцу **«Занятость (%)»** — найдите наиболее/наименее востребованные объекты.
5. Столбец **«Календарь 60 дней»** позволяет визуально оценить загрузку: `111` — занятые дни подряд, `000` — свободные.
6. Отсортируйте по столбцу **«Средняя цена (руб./сут.)»** — сравните стоимость объектов между собой.
7. Столбец **«Цены 60 дней (руб./сут.)»** содержит цены через «;» для каждого дня: `5460;0;5200;5200;0;0;4800;...` Значение `0` означает, что день занят. Это позволяет отследить динамику цен по дням (выходные, праздники, сезонность).

## Внутреннее API sutochno.ru

Программа использует внутреннее JSON API сайта sutochno.ru для получения данных о ценах и занятости. Все запросы выполняются через `fetch()` в контексте браузера, что сохраняет сессию, cookies и обходит CORS-ограничения.

### Сессионный токен

Каждый запрос к API требует заголовок `token` — сессионный идентификатор, который генерируется сайтом при загрузке страницы. Программа перехватывает токен через **route interception** (`page.route("**/api/json/**")`) — перехватываются исходящие запросы к API и извлекается заголовок `token`. Если токен не перехвачен или протух — страница перезагружается и процесс повторяется.

### Эндпоинты

#### `POST /api/json/objects/getPricesAndAvailabilities`

Основной эндпоинт для получения цены и статуса занятости объекта на конкретные даты.

**Заголовки запроса:**

| Заголовок | Значение |
|---|---|
| `Content-Type` | `application/json` |
| `Accept` | `application/json` |
| `token` | (сессионный токен) |
| `platform` | `js` |
| `api-version` | `1.13` |

**Тело запроса:**
```json
{
  "objects": [709383],
  "rooms_cnt": {},
  "guests": 2,
  "date_begin": "2026-05-11 14:00:00",
  "date_end": "2026-05-12 11:00:00",
  "currency_id": 1,
  "is_pets": 0,
  "documents": 0,
  "target": 0,
  "ages": [],
  "no_time": 1
}
```

**Параметры тела запроса:**

| Параметр | Тип | Описание |
|---|---|---|
| `objects` | `array[int]` | Массив ID объявлений |
| `rooms_cnt` | `object` | Количество комнат (пустой объект для стандартного запроса) |
| `guests` | `int` | Количество гостей |
| `date_begin` | `string` | Дата и время заезда (формат: `YYYY-MM-DD HH:MM:SS`) |
| `date_end` | `string` | Дата и время выезда (формат: `YYYY-MM-DD HH:MM:SS`) |
| `currency_id` | `int` | ID валюты (1 = RUB) |
| `is_pets` | `int` | Животные (0 = нет) |
| `documents` | `int` | Отчётные документы (0 = не нужны) |
| `target` | `int` | Цель поездки (0 = не указана) |
| `ages` | `array` | Возраст детей (пустой массив = без детей) |
| `no_time` | `int` | Игнорировать время заезда/выезда (1 = да) |

**Ответ (успех):**
```json
{
  "success": true,
  "data": {
    "objects": [
      {
        "success": true,
        "data": {
          "price": 5040,
          "detail": [
            {
              "date_begin": "2026-05-11 00:00:00",
              "date_end": "2026-05-14 23:59:59",
              "cost": 5040,
              "cost_old": null,
              "nights": 1,
              "type": "season_price"
            }
          ],
          "busy": "unbusy",
          "rooms_available": 4,
          "is_booking_now": true,
          "max_guests": 3,
          "cnt_guests": 2,
          "price_default": 5000,
          "service_fee": 1008,
          "bonus": 202
        },
        "id": 709383
      }
    ]
  }
}
```

**Поля ответа:**

| Поле | Описание |
|---|---|
| `data.objects[].data.price` | Общая стоимость за указанный период |
| `data.objects[].data.detail[]` | Разбивка цен по сезонным периодам |
| `data.objects[].data.detail[].cost` | Цена за одну ночь в данном периоде |
| `data.objects[].data.detail[].type` | Тип цены: `"season_price"` (сезонная) или `1` (базовая) |
| `data.objects[].data.detail[].date_begin` | Начало ценового периода |
| `data.objects[].data.detail[].date_end` | Конец ценового периода |
| `data.objects[].data.busy` | Статус занятости: `"unbusy"` = свободен, `"busy"` = занят |
| `data.objects[].data.rooms_available` | Количество доступных номеров/комнат |
| `data.objects[].data.is_booking_now` | Доступно ли быстрое бронирование |
| `data.objects[].data.max_guests` | Максимальное количество гостей |
| `data.objects[].data.service_fee` | Сервисный сбор |

#### `POST /api/json/objects/calculateBookingPrice`

Расчёт итоговой стоимости бронирования (используется сайтом, программой не вызывается).

**Тело запроса:**
```json
{
  "objectId": 709383,
  "dateBegin": "2026-05-11 14:00:00",
  "dateEnd": "2026-05-12 11:00:00",
  "guests": 2,
  "currencyId": 1,
  "childAges": [],
  "conditions": []
}
```

**Ответ:**
```json
{
  "success": true,
  "data": {
    "calculation": {
      "cost": 5040,
      "costBase": 5040,
      "pricePerDay": 5040,
      "precost": 1008,
      "surcharges": [],
      "discounts": []
    },
    "cntGuests": 2,
    "cntNights": 1,
    "cashback": 202
  }
}
```

**Поля ответа:**

| Поле | Описание |
|---|---|
| `data.calculation.cost` | Итоговая стоимость |
| `data.calculation.pricePerDay` | Средняя цена за ночь |
| `data.calculation.precost` | Предоплата |
| `data.cntNights` | Количество ночей |
| `data.cashback` | Кэшбэк в рублях |

### Как программа использует API

1. Загружает страницу карточки в браузере и перехватывает токен через `page.route("**/api/json/**")`.
2. Валидирует токен тестовым запросом на 2 ночи.
3. Отправляет **bulk-запрос** на 60 ночей (`date_begin = сегодня 14:00`, `date_end = сегодня+60 11:00`).
4. Из ответа bulk-запроса:
   - Извлекает цены из `detail[]` с приоритетом: `season_price` (с датами) → `type=1` (базовая цена).
   - Если `busy == "unbusy"` — все 60 дней свободны, цены готовы.
   - Если `busy == "busy"` — запускает скользящее окно.
5. **Скользящее окно** — для каждого из 60 дней: `date_begin = день 14:00`, `date_end = день+nights 11:00`. Запросы группируются в пакеты по 5 и выполняются параллельно через `Promise.all` в `page.evaluate()`. Пауза 0.5 секунды между пакетами.
6. При массовых ошибках (>30 из 60) — перезагрузка страницы с повторным перехватом токена.
7. Ошибки `min_nights` трактуются как «свободен» (ограничение бронирования, а не фактическая занятость).

## Принципы архитектуры

Проект построен на принципах SOLID:

- **Single Responsibility.** Каждый модуль отвечает за одну задачу: `browser_service` — за браузер, `scraper_service` — за обход каталога, `listing_service` — оркестратор обогащения карточек (делегирует `page_loader`, `token_manager`, `api_client`, `hybrid_strategy`, `enrich_strategies`), `export_service` — за Excel.

- **Open/Closed.** Компоненты расширяемы без модификации. Например, для добавления парсинга цен достаточно расширить `ListingService`, не меняя другие сервисы.

- **Liskov Substitution.** Репозиторий определён как абстрактный класс (`BaseListingRepository`). SQLite-реализация может быть заменена на PostgreSQL без изменения сервисов.

- **Interface Segregation.** Каждый интерфейс содержит только необходимые методы. Сервисы не зависят от лишних методов других модулей.

- **Dependency Inversion.** Сервисы зависят от абстракций. Все зависимости инжектируются через конструкторы.

### Используемые паттерны

- **Repository** — абстракция над хранилищем данных (SQLite).
- **Service Layer** — бизнес-логика в сервисах, координация между компонентами.
- **Factory** — фабричные функции в `__main__.py` для создания зависимостей.
- **Strategy** — `HybridStrategy` внутри `listing/` реализует стратегию получения данных через внутренний JSON API. При смене площадки достаточно заменить модуль `listing/`, не трогая `scraper_service` и `export_service`.
- **Decorator** — retry-декоратор для повторных попыток при сбоях.

## Устранение неполадок

### Токен API не найден

В логах: `«токен_не_перехвачен_при_загрузке»` или `«токен_не_получен_пропуск_карточки»`. Программа перехватывает токен через `page.route("**/api/json/**")` при загрузке страницы карточки. Если токен не перехвачен — возможные причины:

- Страница не загрузилась полностью (проверьте интернет-соединение).
- Сайт изменил механизм авторизации API-запросов.
- Антибот-защита заблокировала сессию.

**Решение:** убедитесь, что `HEADLESS_MODE=false` и страница открывается нормально. Попробуйте увеличить `MIN_DELAY_MS` до 3000–5000. При ошибке программа автоматически перезагружает страницу и пытается получить новый токен.

---

### Ошибка «Обязательная переменная окружения не задана»

Убедитесь, что файл `.env` существует и содержит заполненное значение `SUTOCHNO_SEARCH_URL`. Сверьтесь с `.env.example`.

---

### Цены не собираются (все 0 при свободных днях)

Программа получает цены через внутреннее API sutochno.ru (`getPricesAndAvailabilities`). Если все цены = 0 при свободных днях — возможные причины:

- Сессионный токен API не найден (в логах: `«токен_не_получен_пропуск_карточки»`). Убедитесь, что страница полностью загружается.
- Bulk-запрос вернул ошибку (в логах: `«bulk_запрос_ошибка»`). Проверьте логи на уровне DEBUG.
- Rate-limit со стороны сайта — увеличьте `MIN_DELAY_MS`, `MAX_DELAY_MS` и `TAB_DELAY_MS`.

---

### Прокси не работают / все отмечены как нерабочие

При проверке каждая прокси открывает браузер и загружает sutochno.ru с ожиданием 15 секунд. Если страница не загрузилась — прокси считается нерабочей. Возможные причины: прокси истекли, заблокированы сайтом, или недоступны из вашей сети.

Проверьте формат файла `data/proxies.txt` (формат: `ip:port:login:password`, по одной на строку). Если все прокси нерабочие — программа автоматически переключается в режим без прокси (вкладки или последовательная обработка).

---

### Браузер не запускается

Убедитесь, что Chromium установлен:

```bash
playwright install chromium
```

Если вы на Linux-сервере без графического интерфейса, установите `HEADLESS_MODE=true` в `.env`.

---

### Программа работает медленно

- Уменьшите `MIN_DELAY_MS` и `MAX_DELAY_MS` (но не ниже 1000 — риск блокировки).
- Ограничьте количество страниц через `MAX_PAGES`.
- Каждая карточка обрабатывается за 10–15 секунд (bulk + скользящее окно пакетами по 5). Время обработки каждой карточки выводится в консоль.
- Включите параллельные вкладки (`MAX_TABS=5` или больше) — несколько карточек обрабатываются одновременно в одном браузере.
- Для максимального ускорения включите прокси (`USE_PROXY=true`) с вкладками — каждый прокси-браузер открывает `MAX_TABS` вкладок. Например, 5 прокси × 5 вкладок = до 25 карточек одновременно.

---

### Вкладки падают с ошибками

При `MAX_TABS > 1` вкладки конкурируют за ресурсы. Если наблюдаются частые ошибки:

- Уменьшите `MAX_TABS` до 3.
- Увеличьте `TAB_DELAY_MS` до 5000–7000 мс.
- Убедитесь, что `NAVIGATION_TIMEOUT` не менее 60000 мс.

---

### Excel-файл не создаётся

Убедитесь, что этап парсинга прошёл успешно — без собранных объявлений экспорт пропускается. Проверьте логи на наличие ошибок.

---

### Дубликаты в результатах

Программа автоматически дедуплицирует объявления по ID в процессе парсинга. При повторном запуске существующие записи обновляются (upsert по `external_id`).

---

### Повторный запуск

- База данных сохраняется между запусками. Повторный парсинг обновляет существующие объявления (по `external_id`) и добавляет новые.
- Excel-файл перезаписывается при каждом запуске и содержит все объявления из базы.
- Для чистого старта удалите файл `data/sutochno_listings.db`.
