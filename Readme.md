# Sutochno Parser — мониторинг посуточной аренды

Программа для автоматического сбора объявлений посуточной аренды с сайта sutochno.ru, извлечения данных о ценах, календаре занятости и характеристиках объектов с экспортом результатов в Excel.

---

## Что делает программа

Программа работает в три этапа:

**Этап 1 — Парсинг каталога.** Открывает браузер, переходит по ссылке на страницу поиска sutochno.ru (с уже настроенными фильтрами), обходит все страницы пагинации и собирает базовые данные каждого объявления: ID, название, цену за сутки, рейтинг, количество отзывов, площадь, количество гостей, адрес, ближайшее метро, наличие быстрого бронирования. Дедупликация по ID выполняется автоматически.

**Этап 2 — Парсинг карточек.** Заходит в каждую карточку объявления и извлекает календарь занятости на 60 дней через интерактивный датепикер: открывает календарь, сбрасывает даты, считывает свободные и занятые дни, листает месяцы для покрытия полного диапазона. Затем для каждого свободного дня программа выбирает даты заезда и выезда, считывает актуальную цену за сутки и формирует массив цен на 60 дней.

**Этап 3 — Экспорт.** Формирует Excel-файл с таблицей, автофильтрами и кликабельными ссылками. Вы можете отсортировать объявления по цене, занятости или рейтингу и сравнить объекты между собой.

## Требования

- **Python** 3.11 или выше
- **Операционная система:** Windows, macOS или Linux
- **Стабильное интернет-соединение** для парсинга sutochno.ru

## Установка

### 1. Клонируйте или скачайте проект

Поместите папку `sutochno_parser` в удобное место на вашем компьютере.

### 2. Откройте терминал в папке проекта

cd путь/к/sutochno_parser

3. Создайте виртуальное окружение (рекомендуется)
python -m venv venv

Активируйте его:

venv\Scripts\activate

macOS / Linux:

source venv/bin/activate

4. Установите зависимости

Основные зависимости:
pip install -e 

Для разработки (линтер, типы, тесты):

pip install -e ".[dev]"

5. Установите браузер для Playwright

playwright install chromium

Эта команда скачает Chromium (~150 МБ). Загрузка выполняется один раз.

Настройка
1. Создайте файл .env
Скопируйте шаблон:

Windows:

copy .env.example .env

macOS / Linux:

cp .env.example .env

2. Заполните обязательные переменные

Откройте файл .env в любом текстовом редакторе и заполните:

# Максимальное количество страниц для парсинга (0 = все страницы)

# Минимальная пауза между действиями (мс) — защита от детекции бота

# Максимальная пауза между действиями (мс)
MAX_DELAY_MS=5000

# Уровень логирования (DEBUG, INFO, WARNING, ERROR, CRITICAL)

Запуск
Основная команда

python -m src

Что происходит при запуске
Программа загружает настройки из .env и проверяет обязательные переменные.

Открывается браузер Chromium со stealth-настройками (обход антибот-защиты).

Браузер переходит на указанную страницу поиска sutochno.ru.

Программа собирает базовые данные всех объявлений на каждой странице каталога.

При наличии пагинации — автоматически нажимает «Далее» и переходит на следующие страницы.

После обхода каталога — программа заходит в каждую карточку объявления, извлекает календарь занятости на 60 дней через датепикер, а затем собирает цены за сутки для каждого свободного дня (для занятых дней цена = 0). Если включён режим прокси (USE_PROXY=true) — карточки обрабатываются параллельно через несколько браузеров, каждый со своей прокси, что значительно ускоряет сбор данных.

Собранные данные сохраняются в SQLite (data/sutochno_listings.db).

Результаты экспортируются в Excel (data/sutochno_report.xlsx).

Логи сохраняются в файл logs/app.log в формате JSON с ротацией (10 МБ × 5 копий).

Альтернативный запуск (если установлена как пакет)

sutochno-parser

Как работать с файлом
Откройте файл в Excel или LibreOffice Calc.

Используйте автофильтр — выберите нужные параметры (цена, рейтинг, район).

Отсортируйте по столбцу «Цена (руб./сут.)» — увидите объекты от дешёвых к дорогим.

Отсортируйте по столбцу «Занятость (%)» — найдите наиболее/наименее востребованные объекты.

Столбец «Календарь 60 дней» позволяет визуально оценить загрузку: 111 — занятые дни подряд, 000 — свободные.

Отсортируйте по столбцу «Средняя цена (руб./сут.)» — сравните стоимость объектов между собой.

Столбец «Цены 60 дней (руб./сут.)» содержит цены через «;» для каждого дня: 5460;0;5200;5200;0;0;4800;... Значение 0 означает, что день занят. Это позволяет отследить динамику цен по дням (выходные, праздники, сезонность).

Принципы архитектуры
Проект построен на принципах SOLID:

Single Responsibility. Каждый модуль отвечает за одну задачу: browser_service — за браузер, scraper_service — за обход каталога, listing_service — за парсинг карточки, export_service — за Excel.

Open/Closed. Компоненты расширяемы без модификации. Например, для добавления парсинга цен достаточно расширить ListingService, не меняя другие сервисы.

Liskov Substitution. Репозиторий определён как абстрактный класс (BaseListingRepository). SQLite-реализация может быть заменена на PostgreSQL без изменения сервисов.

Interface Segregation. Каждый интерфейс содержит только необходимые методы. Сервисы не зависят от лишних методов других модулей.

Dependency Inversion. Сервисы зависят от абстракций. Все зависимости инжектируются через конструкторы.

Используемые паттерны
Repository — абстракция над хранилищем данных (SQLite).

Service Layer — бизнес-логика в сервисах, координация между компонентами.

Factory — фабричные функции в __main__.py для создания зависимостей.

Strategy — ListingService можно заменить на альтернативную реализацию для другой площадки.

Decorator — retry-декоратор для повторных попыток при сбоях.

Устранение неполадок
Ошибка «Обязательная переменная окружения не задана»
Убедитесь, что файл .env существует и содержит заполненное значение SUTOCHNO_SEARCH_URL. Сверьтесь с .env.example.

Цены не собираются (все 0 при свободных днях)

Программа считывает цену из двух мест на странице: основной блок цены и альтернативный (для отелей). Если цена не найдена ни в одном — в логах появится предупреждение «цена_не_найдена_ни_в_одном_селекторе». Возможные причины: изменение вёрстки сайта, слишком быстрые действия (увеличьте MIN_DELAY_MS). Проверьте логи на уровне DEBUG для деталей.

Прокси не работают / все отмечены как нерабочие

При проверке каждая прокси открывает браузер и загружает sutochno.ru с ожиданием 15 секунд. Если страница не загрузилась — прокси считается нерабочей. Возможные причины: прокси истекли, заблокированы сайтом, или недоступны из вашей сети. Проверьте формат файла data/proxies.txt (ip:port:login:password, по одной на строку). Если все прокси нерабочие — программа автоматически переключается в последовательный режим без прокси.

Браузер не запускается
Убедитесь, что Chromium установлен:

playwright install chromium

Если вы на Linux-сервере без графического интерфейса, установите HEADLESS_MODE=true в .env.

Программа работает медленно

Уменьшите MIN_DELAY_MS и MAX_DELAY_MS (но не ниже 1000 — риск блокировки).

Ограничьте количество страниц через MAX_PAGES.

Учитывайте, что программа заходит в каждую карточку объявления и для каждого свободного дня из 60 выполняет цикл «открыть календарь → выбрать даты → считать цену». При 40 свободных днях одна карточка обрабатывается ~2–3 минуты. Время обработки каждой карточки выводится в консоль.

Для ускорения включите режим прокси (USE_PROXY=true) — карточки будут обрабатываться параллельно через несколько браузеров. Например, 5 прокси обрабатывают 50 карточек в 5 раз быстрее (каждая прокси получает по 10 карточек).

Календарь не загружается в карточке
Программа пытается открыть датепикер и считать данные. Если элементы не найдены — карточка пропускается с предупреждением в логах, остальные объявления обрабатываются нормально.

Excel-файл не создаётся
Убедитесь, что этап парсинга прошёл успешно — без собранных объявлений экспорт пропускается. Проверьте логи на наличие ошибок.

Дубликаты в результатах
Программа автоматически дедуплицирует объявления по ID в процессе парсинга. При повторном запуске существующие записи обновляются (upsert по external_id).

Повторный запуск

База данных сохраняется между запусками. Повторный парсинг обновляет существующие объявления (по external_id) и добавляет новые.

Excel-файл перезаписывается при каждом запуске и содержит все объявления из базы.

Для чистого старта удалите файл data/sutochno_listings.db.
